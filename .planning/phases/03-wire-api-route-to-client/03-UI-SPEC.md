## MessageInput Fetch Integration

### `handleSubmit` Flow

```
User presses Enter / Send
    ↓
Validate: trimmed !== '' AND NOT isLoading AND NOT isSubmittingRef
    ↓
Set isSubmittingRef = true  (synchronous guard)
addMessage({ role: 'user', content: trimmed })
setLoading(true)
setInput('')
    ↓
POST /api/ai-chatbot
  headers: { 'Content-Type': 'application/json' }
  body: { message, messages: currentStoreMessages }
    ↓
┌─ HTTP 2xx + type !== 'error' ─────────────────────┐
│ addMessage({ role: 'assistant', content: data.reply })  │
└─────────────────────────────────────────────────────┘
┌─ HTTP non-2xx OR type === 'error' ─────────────────┐
│ addMessage({ role: 'assistant', content: errorMsg, type: 'error' })
└────────────────────────────────────────────────────┘
┌─ Network exception ──────────────────────────────────┐
│ addMessage({ role: 'assistant', content: 'Connection error. Please check your network and try again.', type: 'error' })
└─────────────────────────────────────────────────────┘
    ↓
setLoading(false)
isSubmittingRef = false
```

**Critical ordering:** `setLoading(false)` MUST be in `finally` block — guaranteed to run even if `addMessage` throws (unlikely but defensive).

**Check order for error detection:** Always check `!res.ok` BEFORE checking `data.type === 'error'`. HTTP error responses (502, 500) do not include the `type` field — they only include `error`.

  "reply": "Bot's response text",

  "role": "CUSTOMER",

  ]
