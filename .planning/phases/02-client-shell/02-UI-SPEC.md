## Spacing Scale

All values are multiples of 4 (8-point grid baseline from Tailwind project standard).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, tight padding |
| sm | 8px | Compact element spacing, bubble gap |
| md | 16px | Default element spacing, modal padding |
| lg | 24px | Section padding, header padding |
| xl | 32px | FAB offset from edges |
| 2xl | 48px | Major section breaks |

**Exceptions:** None for Phase 2. All touch targets Ōēź 44px via Button `size="icon"` (32├Ś32 minimum, `size-8` = 2rem/32px for FAB interior controls).

### Props & States Per Component

**`FloatingFAB`**
- Reads `isOpen` from `useAIChatbotStore`
- `hidden` class when `isOpen === true`
- `aria-label="Open AI Assistant"`
- On click: `setOpen(true)`

**`AIChatbotModal`**
- Reads `isOpen` from store
- Uses existing `Dialog` / `DialogContent` from `@/components/ui/dialog`
- On close (X / Escape / backdrop): `setOpen(false)` ŌåÆ calls `clearMessages()`
- Closes on route change via `usePathname` listener
- Focus trap handled by `@base-ui/react/dialog`
- First focusable element: close button (handled by existing dialog)

**`useAIChatbotStore`**
```ts
interface Message {
  id: string
  role: 'user' | 'assistant'
  content: string
  timestamp: number
}

interface AIChatbotStore {
  messages: Message[]
  isLoading: boolean
  isOpen: boolean
  addMessage: (message: Omit<Message, 'id' | 'timestamp'>) => void
  setLoading: (loading: boolean) => void
  setOpen: (open: boolean) => void
  clearMessages: () => void
}
```
No persistence middleware.

**`MessageList`**
- Auto-scrolls to bottom via `useRef` + `scrollIntoView` on new message
- Empty state: initial bot greeting bubble
- Renders `MessageBubble` for each message + `LoadingIndicator` after last message when `isLoading`

**`MessageBubble`**
- `role === 'user'`: right-aligned, coral bg, white text
- `role === 'assistant'`: left-aligned, muted bg, dark text + bot avatar
- `react-markdown` + `remark-gfm` for bot messages (bold, italic, lists, tables)
- User messages: plain text (no markdown)
- `aria-label` on bubble: `"User message"` / `"AI Assistant response"`

**`MessageInput`**
- `Input` component from `@/components/ui/input`
- `Send` (or `ArrowUp`) icon button from `lucide-react`
- Enter key submits
- Disabled + opacity-50 when `isLoading`
- Clears input on submit
- Calls `addMessage` + `setLoading(true)` on submit
- Placeholder: `Ask about your contracts or training sessions...`

**`LoadingIndicator`**
- Renders only when `isLoading === true`
- Three `div` dots with CSS bounce animation
- Label: "GoChul is thinking..."

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| FAB click | Opens modal, hides FAB |
| Modal X button | Closes modal, clears messages, shows FAB |
| Escape key | Closes modal, clears messages, shows FAB |
| Backdrop click | Closes modal, clears messages, shows FAB |
| Route change | Closes modal, clears messages, shows FAB |
| Enter key in input | Submits message (same as send button) |
| Send button click | Submits message |
| Input while loading | Disabled |
| New user message | Appears immediately as coral bubble, auto-scrolls |
| New bot message | Appears as muted bubble with avatar, auto-scrolls |
